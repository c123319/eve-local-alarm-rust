<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Unbounded frame queue causes memory growth during active monitoring | medium | Implement latest-frame-wins with a single-slot shared frame or bounded channel of capacity 1. |
| Capture worker survives after stop and keeps screen capture active | medium | Store cancellation flag and `JoinHandle`; stop must set cancellation and join before reporting stopped. |
| Invalid ROI dimensions cause capture panic or native errors | medium | Validate `width > 0`, `height > 0`, FPS range, and monitor bounds before starting worker. |
| Screen image data accidentally emitted to frontend too often | low | Plan 02-01 exposes Rust-side frame contract only; do not stream frame buffers over Tauri events in this plan. |
</threat_model>

  <action>
Create the capture module and stable frame contract:

1. Create `src-tauri/src/capture/mod.rs`:
   ```rust
   pub mod mss;

   pub use mss::{CapturedFrame, MonitoringSnapshot, MonitoringStatus, MssCaptureWorker};
   ```

2. Create `src-tauri/src/capture/mss.rs` with these public types:
   - `CapturedFrame { roi_id: String, region: Rect, captured_at_ms: u128, width: u32, height: u32, rgba: Vec<u8> }`
   - `MonitoringStatus` enum with `Idle`, `Starting`, `Running`, `Stopping`, `Stopped`, `Error`.
   - `MonitoringSnapshot { status: MonitoringStatus, last_error: Option<String>, capture_fps: u32, last_frame_at_ms: Option<u128> }`.
   - `MssCaptureWorker` owning `Arc<AtomicBool>` cancellation and `Option<std::thread::JoinHandle<()>>`.

3. Implement helper functions:
   - `validate_capture_fps(fps: u32) -> Result<u32, String>` allowing `1..=30` and returning an error containing `捕获帧率必须在 1 到 30 FPS 之间` for invalid values.
   - `validate_roi(region: &Rect) -> Result<(), String>` requiring `width > 0` and `height > 0` and returning an error containing `ROI 宽度和高度必须大于 0` for invalid values.
   - `now_millis() -> u128` for frame timestamps.

4. Implement worker start logic using `std::thread::spawn`, `std::sync::{Arc, Mutex, atomic::{AtomicBool, Ordering}}`, and latest-frame-wins storage with `Arc<Mutex<Option<CapturedFrame>>>`.

5. Inside the worker loop, use XCap monitor region capture in the MSS path:
   ```rust
   let monitors = xcap::Monitor::all().map_err(|e| format!("获取显示器列表失败: {e}"))?;
   let monitor = monitors.into_iter().next().ok_or_else(|| "未找到可用显示器".to_string())?;
   let image = monitor.capture_region(region.x, region.y, region.width, region.height)
       .map_err(|e| format!("MSS 区域捕获失败: {e}"))?;
   ```
   Convert the returned image into RGBA bytes for `CapturedFrame.rgba` using the image API exposed by XCap.

6. Add `mod capture;` to `src-tauri/src/lib.rs`.
  </action>
