<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| User double-clicks start and spawns duplicate workers | medium | Backend state rejects `start_monitoring` while status is Starting or Running; UI disables start button. |
| Stop returns before worker is stopped | medium | `stop_monitoring` sets cancellation and joins worker before returning stopped status. |
| Frontend status diverges from backend truth | low | UI initializes from `get_monitoring_status` and updates through `monitoring-status` events. |
| Phase scope creep into ROI/detection/alerts | low | UI only adds lifecycle controls and warning copy; no preview/detection/alert/tray controls. |
</threat_model>

Update `src/components/SettingsPanel.tsx` with the Phase 2 monitoring UI contract:

1. Import `listen` from `@tauri-apps/api/event`.

2. Add TypeScript types:
   ```ts
   type MonitoringStatus = 'Idle' | 'Starting' | 'Running' | 'Stopping' | 'Stopped' | 'Error';

   interface MonitoringSnapshot {
     status: MonitoringStatus;
     last_error: string | null;
     capture_fps: number;
     last_frame_at_ms: number | null;
   }
   ```

3. Add React state for monitoring snapshot and error text. Initialize by invoking `get_monitoring_status` in `useEffect`.

4. Add `listen<MonitoringSnapshot>('monitoring-status', ...)` and `listen<string>('monitoring-error', ...)` in `useEffect`; call both `unlisten` cleanup functions on unmount.

5. Add `handleStartMonitoring`:
   - Set local status to Starting before invoke.
   - Call `await invoke<MonitoringSnapshot>('start_monitoring', { config })`.
   - Update snapshot from returned value.
   - On error, show Chinese error message `监控失败：{error}。请检查配置后重试。`.

6. Add `handleStopMonitoring`:
   - Confirm with `window.confirm('停止监控：确定要停止当前监控吗？')`.
   - Set local status to Stopping before invoke.
   - Call `await invoke<MonitoringSnapshot>('stop_monitoring')`.
   - Update snapshot from returned value.

7. Render a new section titled `监控控制` containing:
   - Label `当前状态：` and Chinese state text mapping: `Idle -> 未启动`, `Starting -> 启动中...`, `Running -> 运行中`, `Stopping -> 停止中...`, `Stopped -> 已停止`, `Error -> 错误`.
   - Warning text exactly `⚠️ MSS 模式仅捕获屏幕可见区域。请确保监控区域不被遮挡。`.
   - FPS readout `捕获帧率：{capture_fps} FPS`.
   - Buttons `开始监控` and `停止监控` with disabled states matching `02-UI-SPEC.md`.
   - Error box containing `监控失败：` when `last_error` or local error exists.
   - Status container with `aria-live="polite"`.

8. Use darker orange `#e67e22` for Starting/Stopping status per UI-SPEC accessibility note.
